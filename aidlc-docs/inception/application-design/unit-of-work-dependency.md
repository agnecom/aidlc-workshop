# Unit of Work Dependencies

## 의존성 매트릭스

| | Unit 1 (인증) | Unit 2 (메뉴) | Unit 3 (주문) | Unit 4 (테이블) |
|---|---|---|---|---|
| **Unit 1 (인증)** | - | 없음 | 없음 | 없음 |
| **Unit 2 (메뉴)** | 의존 (인증 미들웨어, DB 모델) | - | 없음 | 없음 |
| **Unit 3 (주문)** | 의존 (인증, DB 모델, 테이블 세션) | 의존 (메뉴 데이터) | - | 없음 |
| **Unit 4 (테이블)** | 의존 (인증, DB 모델) | 없음 | 의존 (주문 데이터, SSEManager) | - |

## 개발 순서

```
Unit 1 (인증/기반)
  └──→ Unit 2 (메뉴)
         └──→ Unit 3 (주문/SSE)
                └──→ Unit 4 (테이블 관리)
```

**순서 근거:**
1. **Unit 1 먼저**: 프로젝트 구조, DB 모델, 인증이 모든 후속 단위의 기반
2. **Unit 2 다음**: 메뉴 데이터가 있어야 주문 생성 가능
3. **Unit 3 다음**: 주문/SSE가 있어야 테이블 관리(주문 삭제, 이용 완료) 가능
4. **Unit 4 마지막**: 앞선 모든 기능에 의존

## 통합 포인트

| 통합 포인트 | 관련 Unit | 설명 |
|------------|-----------|------|
| JWT 인증 미들웨어 | 1 → 2, 3, 4 | 모든 보호된 API에서 사용 |
| DB 모델/스키마 | 1 → 2, 3, 4 | Unit 1에서 전체 모델 정의 |
| SSEManager | 3 → 4 | 주문 이벤트 브로드캐스트 공유 |
| 테이블 세션 | 1, 4 → 3 | 주문 생성 시 세션 ID 참조 |
