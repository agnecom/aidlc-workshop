# 테이블오더 서비스 요구사항

## Intent Analysis

- **사용자 요청**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **요청 유형**: New Project (신규 프로젝트)
- **범위**: Multiple Components (고객 UI, 관리자 UI, Backend API, DB)
- **복잡도**: Complex (실시간 통신, 인증, 파일 업로드, 다중 사용자 유형)

---

## 기술 스택 결정사항

| 영역 | 기술 | 비고 |
|------|------|------|
| Backend | Python (FastAPI) | REST API + SSE |
| Frontend | Vue.js | 고객용 + 관리자용 |
| Database | PostgreSQL | 관계형 DB |
| 배포 | Docker Compose | 로컬 개발 환경 |
| 이미지 관리 | 서버 업로드 | 로컬 파일 시스템 저장 |

## 프로젝트 범위 결정사항

| 항목 | 결정 |
|------|------|
| 매장 지원 | 단일 매장 (MVP) |

## 기능 요구사항 (Functional Requirements)

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 매장 식별자, 테이블 번호, 테이블 비밀번호로 초기 설정 (1회)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

### FR-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시 → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-05: 주문 내역 조회 (고객)
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- SSE 기반 주문 상태 실시간 업데이트

### FR-06: 관리자 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션
- bcrypt 비밀번호 해싱
- 로그인 시도 제한
- 관리자 계정은 seed 데이터로 생성

### FR-07: 실시간 주문 모니터링 (관리자)
- SSE 기반 실시간 주문 목록 업데이트
- 테이블별 그리드/카드 레이아웃
- 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

### FR-08: 테이블 관리 (관리자)
- 테이블 초기 설정 (번호, 비밀번호, 세션 생성)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 테이블 세션 종료/이용 완료 (주문 내역 → 과거 이력 이동, 현재 목록 리셋)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

### FR-09: 메뉴 관리 (관리자)
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지)
- 이미지 파일 서버 업로드
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 통신 (주문 표시 2초 이내)
- 소규모 환경 최적화 (테이블 10개 이하)

### NFR-02: 보안
- JWT 토큰 기반 인증 (관리자 16시간, 테이블 16시간)
- bcrypt 비밀번호 해싱
- 로그인 시도 제한
- SECURITY-01 ~ SECURITY-15 전체 적용

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 레이아웃
- 명확한 시각적 계층 구조

### NFR-04: 데이터 무결성
- 장바구니 로컬 저장 (새로고침 유지)
- 세션 기반 주문 그룹화
- 주문 이력 영구 보존

### NFR-05: 배포
- Docker Compose 기반 로컬 개발 환경
- Backend + Frontend + PostgreSQL 컨테이너 구성

## 제외 범위

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜, 지도, 번역)

## 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보 (단일 매장)
- **Admin**: 관리자 계정 (seed 데이터)
- **Table**: 테이블 정보 (번호, 비밀번호)
- **TableSession**: 테이블 세션 (시작/종료 시각)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목 (이름, 가격, 설명, 이미지, 순서)
- **Order**: 주문 (테이블, 세션, 상태, 총 금액)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)
