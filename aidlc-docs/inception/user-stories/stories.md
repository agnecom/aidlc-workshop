# User Stories

User Journey-Based 접근법으로 구성. Acceptance Criteria는 Given/When/Then 형식.
에러/예외 시나리오는 별도 스토리(US-ERR-xx)로 분리.

---

## 고객 Journey: 입장 → 메뉴 탐색 → 장바구니 → 주문 → 내역 확인

### US-01: 테이블 자동 로그인

**As a** 고객,
**I want** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를,
**So that** 대기 시간 없이 즉시 주문을 시작할 수 있다.

**Acceptance Criteria:**

1. Given 관리자가 테이블 초기 설정을 완료한 태블릿
   When 태블릿 브라우저에서 서비스에 접속하면
   Then 저장된 인증 정보로 자동 로그인되어 메뉴 화면이 표시된다

2. Given 자동 로그인이 성공한 상태
   When 16시간이 경과하면
   Then 세션이 만료되고 재인증이 필요하다

3. Given 자동 로그인이 성공한 상태
   When 브라우저를 새로고침하면
   Then 세션이 유지되어 메뉴 화면이 다시 표시된다

**관련 요구사항**: FR-01

---

### US-02: 메뉴 조회 및 탐색

**As a** 고객,
**I want** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를,
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**Acceptance Criteria:**

1. Given 메뉴 화면에 접속한 상태
   When 화면이 로드되면
   Then 카테고리 목록과 첫 번째 카테고리의 메뉴가 카드 형태로 표시된다

2. Given 메뉴 목록이 표시된 상태
   When 특정 카테고리를 선택하면
   Then 해당 카테고리의 메뉴만 필터링되어 표시된다

3. Given 메뉴 카드가 표시된 상태
   When 메뉴 카드를 확인하면
   Then 메뉴명, 가격, 설명, 이미지가 표시된다

4. Given 메뉴 화면이 표시된 상태
   When 모든 버튼/터치 영역을 확인하면
   Then 최소 44x44px 크기로 터치 친화적이다

**관련 요구사항**: FR-02

---

### US-03: 장바구니 관리

**As a** 고객,
**I want** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를,
**So that** 주문 전에 선택한 메뉴를 검토하고 수정할 수 있다.

**Acceptance Criteria:**

1. Given 메뉴 화면에서 메뉴를 선택한 상태
   When 메뉴를 장바구니에 추가하면
   Then 장바구니에 해당 메뉴가 추가되고 총 금액이 업데이트된다

2. Given 장바구니에 메뉴가 있는 상태
   When 수량 증가/감소 버튼을 누르면
   Then 수량이 변경되고 총 금액이 실시간으로 재계산된다

3. Given 장바구니에 메뉴가 있는 상태
   When 특정 메뉴를 삭제하면
   Then 해당 메뉴가 장바구니에서 제거되고 총 금액이 재계산된다

4. Given 장바구니에 메뉴가 있는 상태
   When 브라우저를 새로고침하면
   Then 장바구니 내용이 로컬 저장소에서 복원된다

5. Given 장바구니에 메뉴가 있는 상태
   When 장바구니 비우기를 실행하면
   Then 모든 메뉴가 제거되고 총 금액이 0이 된다

**관련 요구사항**: FR-03

---

### US-04: 주문 생성

**As a** 고객,
**I want** 장바구니의 메뉴를 주문으로 확정하기를,
**So that** 매장에 주문이 전달되어 음식을 받을 수 있다.

**Acceptance Criteria:**

1. Given 장바구니에 1개 이상의 메뉴가 있는 상태
   When 주문 확정 버튼을 누르면
   Then 주문 내역(메뉴명, 수량, 단가, 총 금액)이 최종 확인 화면에 표시된다

2. Given 주문 확인 화면에서 확정을 누른 상태
   When 주문이 성공적으로 생성되면
   Then 주문 번호가 표시되고, 장바구니가 비워지고, 메뉴 화면으로 자동 리다이렉트된다

3. Given 주문이 성공한 상태
   When 주문 정보를 확인하면
   Then 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함되어 있다

**관련 요구사항**: FR-04

---

### US-05: 주문 내역 조회

**As a** 고객,
**I want** 현재 테이블 세션의 주문 내역과 상태를 확인하기를,
**So that** 주문이 정상 접수되었는지, 현재 진행 상태가 어떤지 알 수 있다.

**Acceptance Criteria:**

1. Given 주문 내역 화면에 접속한 상태
   When 화면이 로드되면
   Then 현재 테이블 세션의 주문만 시간 순으로 표시된다

2. Given 주문 목록이 표시된 상태
   When 각 주문을 확인하면
   Then 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다

3. Given 주문 목록이 표시된 상태
   When 관리자가 주문 상태를 변경하면
   Then SSE를 통해 실시간으로 상태가 업데이트된다

4. Given 테이블 세션이 종료된 이후
   When 새 세션에서 주문 내역을 조회하면
   Then 이전 세션의 주문은 표시되지 않는다

**관련 요구사항**: FR-05

## 관리자 Journey: 로그인 → 주문 모니터링 → 상태 관리 → 테이블 관리 → 메뉴 관리

### US-06: 관리자 로그인

**As a** 관리자,
**I want** 매장 식별자와 계정 정보로 로그인하기를,
**So that** 매장 관리 시스템에 안전하게 접근할 수 있다.

**Acceptance Criteria:**

1. Given 관리자 로그인 화면
   When 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면
   Then JWT 토큰이 발급되고 관리자 대시보드로 이동한다

2. Given 로그인 성공 상태
   When 16시간이 경과하면
   Then 세션이 만료되고 로그인 화면으로 리다이렉트된다

3. Given 로그인 성공 상태
   When 브라우저를 새로고침하면
   Then 세션이 유지되어 대시보드가 다시 표시된다

**관련 요구사항**: FR-06

---

### US-07: 실시간 주문 모니터링

**As a** 관리자,
**I want** 들어오는 주문을 실시간으로 테이블별 대시보드에서 확인하기를,
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다.

**Acceptance Criteria:**

1. Given 관리자 대시보드에 접속한 상태
   When 화면이 로드되면
   Then 테이블별 그리드/카드 레이아웃으로 현재 주문 현황이 표시된다

2. Given 대시보드가 표시된 상태
   When 고객이 새 주문을 생성하면
   Then 2초 이내에 SSE를 통해 해당 테이블 카드에 주문이 표시되고 시각적으로 강조된다

3. Given 테이블 카드가 표시된 상태
   When 카드를 확인하면
   Then 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다

4. Given 테이블 카드가 표시된 상태
   When 카드를 클릭하면
   Then 해당 테이블의 전체 주문 메뉴 목록 상세가 표시된다

**관련 요구사항**: FR-07

---

### US-08: 주문 상태 변경

**As a** 관리자,
**I want** 주문 상태를 대기중 → 준비중 → 완료로 변경하기를,
**So that** 고객에게 주문 진행 상황을 알릴 수 있다.

**Acceptance Criteria:**

1. Given 주문 상세 화면에서 대기중 상태의 주문
   When 상태를 "준비중"으로 변경하면
   Then 주문 상태가 업데이트되고 고객 화면에 SSE로 실시간 반영된다

2. Given 준비중 상태의 주문
   When 상태를 "완료"로 변경하면
   Then 주문 상태가 업데이트되고 고객 화면에 SSE로 실시간 반영된다

**관련 요구사항**: FR-07

---

### US-09: 테이블 초기 설정

**As a** 관리자,
**I want** 테이블 번호와 비밀번호를 설정하기를,
**So that** 고객 태블릿이 자동 로그인할 수 있도록 준비할 수 있다.

**Acceptance Criteria:**

1. Given 테이블 관리 화면
   When 테이블 번호와 비밀번호를 입력하고 설정을 완료하면
   Then 테이블이 생성되고 16시간 세션이 시작된다

2. Given 테이블 설정이 완료된 상태
   When 해당 테이블의 태블릿에서 서비스에 접속하면
   Then 설정된 정보로 자동 로그인이 가능하다

**관련 요구사항**: FR-08

---

### US-10: 테이블 세션 관리 (주문 삭제 및 이용 완료)

**As a** 관리자,
**I want** 테이블의 주문을 삭제하고 이용 완료 처리를 하기를,
**So that** 테이블 상태를 정확하게 관리하고 새 고객을 받을 수 있다.

**Acceptance Criteria:**

1. Given 테이블에 주문이 있는 상태
   When 특정 주문의 삭제 버튼을 누르면
   Then 확인 팝업이 표시된다

2. Given 삭제 확인 팝업이 표시된 상태
   When 확인을 누르면
   Then 주문이 즉시 삭제되고 테이블 총 주문액이 재계산된다

3. Given 테이블에 주문이 있는 상태
   When 이용 완료 버튼을 누르고 확인하면
   Then 현재 세션의 주문 내역이 과거 이력으로 이동하고, 테이블 주문 목록과 총 주문액이 0으로 리셋된다

4. Given 이용 완료 처리된 테이블
   When 새 고객이 주문을 시작하면
   Then 이전 주문 내역 없이 새 세션으로 시작된다

**관련 요구사항**: FR-08

---

### US-11: 과거 주문 내역 조회 (관리자)

**As a** 관리자,
**I want** 테이블별 과거 주문 내역을 조회하기를,
**So that** 이전 매출과 주문 이력을 확인할 수 있다.

**Acceptance Criteria:**

1. Given 테이블 상세 화면
   When "과거 내역" 버튼을 클릭하면
   Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다

2. Given 과거 내역이 표시된 상태
   When 각 주문을 확인하면
   Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다

3. Given 과거 내역이 표시된 상태
   When 날짜 필터를 적용하면
   Then 해당 날짜 범위의 주문만 필터링되어 표시된다

4. Given 과거 내역이 표시된 상태
   When "닫기" 버튼을 클릭하면
   Then 대시보드로 복귀한다

**관련 요구사항**: FR-08

---

### US-12: 메뉴 관리

**As a** 관리자,
**I want** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정하기를,
**So that** 매장 메뉴를 최신 상태로 유지할 수 있다.

**Acceptance Criteria:**

1. Given 메뉴 관리 화면
   When 새 메뉴 등록 시 메뉴명, 가격, 설명, 카테고리, 이미지를 입력하고 저장하면
   Then 메뉴가 등록되고 고객 화면에 표시된다

2. Given 등록된 메뉴가 있는 상태
   When 메뉴 정보를 수정하고 저장하면
   Then 변경 사항이 즉시 반영된다

3. Given 등록된 메뉴가 있는 상태
   When 메뉴를 삭제하면
   Then 메뉴가 제거되고 고객 화면에서 더 이상 표시되지 않는다

4. Given 메뉴 이미지 등록 시
   When 이미지 파일을 업로드하면
   Then 서버에 저장되고 URL이 자동 생성되어 메뉴에 연결된다

5. Given 카테고리 내 메뉴가 여러 개인 상태
   When 메뉴 노출 순서를 변경하면
   Then 고객 화면에서 변경된 순서로 표시된다

6. Given 메뉴 등록/수정 시
   When 필수 필드가 비어있거나 가격이 범위를 벗어나면
   Then 검증 에러 메시지가 표시되고 저장이 차단된다

**관련 요구사항**: FR-09

## 에러/예외 처리 스토리

### US-ERR-01: 고객 자동 로그인 실패 처리

**As a** 고객,
**I want** 자동 로그인이 실패했을 때 명확한 안내를 받기를,
**So that** 문제 상황을 이해하고 관리자에게 도움을 요청할 수 있다.

**Acceptance Criteria:**

1. Given 저장된 인증 정보가 유효하지 않은 상태
   When 자동 로그인을 시도하면
   Then "인증 정보가 만료되었습니다. 관리자에게 문의해 주세요." 메시지가 표시된다

2. Given 서버에 연결할 수 없는 상태
   When 자동 로그인을 시도하면
   Then "서버에 연결할 수 없습니다. 잠시 후 다시 시도해 주세요." 메시지가 표시된다

**관련 요구사항**: FR-01

---

### US-ERR-02: 주문 생성 실패 처리

**As a** 고객,
**I want** 주문이 실패했을 때 장바구니가 유지되고 에러 안내를 받기를,
**So that** 다시 주문을 시도할 수 있다.

**Acceptance Criteria:**

1. Given 주문 확정을 시도한 상태
   When 서버 오류로 주문이 실패하면
   Then 에러 메시지가 표시되고 장바구니 내용이 유지된다

2. Given 주문 확정을 시도한 상태
   When 네트워크 오류로 주문이 실패하면
   Then "네트워크 오류가 발생했습니다. 다시 시도해 주세요." 메시지가 표시되고 장바구니가 유지된다

**관련 요구사항**: FR-04

---

### US-ERR-03: 고객 SSE 연결 끊김 처리

**As a** 고객,
**I want** 실시간 연결이 끊어졌을 때 자동 재연결되기를,
**So that** 주문 상태 업데이트를 계속 받을 수 있다.

**Acceptance Criteria:**

1. Given SSE 연결이 활성화된 상태
   When 네트워크 문제로 연결이 끊어지면
   Then 자동으로 재연결을 시도한다

2. Given 재연결 시도 중
   When 재연결에 실패하면
   Then "실시간 업데이트가 중단되었습니다. 페이지를 새로고침해 주세요." 메시지가 표시된다

**관련 요구사항**: FR-05

---

### US-ERR-04: 관리자 로그인 실패 처리

**As a** 관리자,
**I want** 로그인 실패 시 명확한 에러 메시지를 받고, 반복 실패 시 보호되기를,
**So that** 계정 보안이 유지되면서도 올바른 정보로 재시도할 수 있다.

**Acceptance Criteria:**

1. Given 관리자 로그인 화면
   When 잘못된 자격 증명으로 로그인을 시도하면
   Then "매장 식별자, 사용자명 또는 비밀번호가 올바르지 않습니다." 메시지가 표시된다

2. Given 연속 로그인 실패 상태
   When 제한 횟수를 초과하면
   Then 일정 시간 동안 로그인 시도가 차단되고 안내 메시지가 표시된다

**관련 요구사항**: FR-06

---

### US-ERR-05: 관리자 SSE 연결 끊김 처리

**As a** 관리자,
**I want** 실시간 모니터링 연결이 끊어졌을 때 자동 재연결되기를,
**So that** 주문을 놓치지 않을 수 있다.

**Acceptance Criteria:**

1. Given SSE 연결이 활성화된 상태
   When 연결이 끊어지면
   Then 자동으로 재연결을 시도한다

2. Given 재연결에 실패한 상태
   When 재연결 시도가 반복 실패하면
   Then "실시간 연결이 끊어졌습니다. 페이지를 새로고침해 주세요." 경고가 표시된다

**관련 요구사항**: FR-07

---

### US-ERR-06: 관리자 작업 실패 처리

**As a** 관리자,
**I want** 주문 삭제, 상태 변경, 메뉴 관리 등의 작업이 실패했을 때 명확한 피드백을 받기를,
**So that** 문제를 파악하고 재시도할 수 있다.

**Acceptance Criteria:**

1. Given 주문 삭제를 시도한 상태
   When 서버 오류로 삭제가 실패하면
   Then 에러 메시지가 표시되고 주문이 유지된다

2. Given 주문 상태 변경을 시도한 상태
   When 변경이 실패하면
   Then 에러 메시지가 표시되고 이전 상태가 유지된다

3. Given 메뉴 등록/수정을 시도한 상태
   When 서버 오류로 저장이 실패하면
   Then 에러 메시지가 표시되고 입력 내용이 유지된다

**관련 요구사항**: FR-07, FR-08, FR-09
